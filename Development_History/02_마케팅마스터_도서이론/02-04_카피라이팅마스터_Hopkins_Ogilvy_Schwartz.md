# 카피라이팅 마스터 도서 이론 완전체

> Claude Hopkins / David Ogilvy / Eugene Schwartz / John Caples / Joe Sugarman / Gary Halbert / Victor Schwab

## 1. Claude Hopkins — Scientific Advertising (1923)

### 핵심 철학
> "광고는 예술이 아니라 과학이다. 모든 광고는 테스트로 검증되어야 한다."

### 핵심 원칙 7가지

| 원칙 | 내용 | 실전 적용 |
|------|------|-----------|
| **쿠폰 테스트** | 어떤 광고가 더 효과적인지 쿠폰 응답률로 측정 | A/B 테스트, 제목 2가지 배포 후 클릭률 비교 |
| **헤드라인 우선** | 헤드라인이 광고비의 90%를 결정 | 제목에 가장 많은 시간 투자 |
| **구체적 수치** | "빨리" 대신 "3일 안에" / "많이" 대신 "47%" | 썸네일/제목에 숫자 삽입 |
| **이유 제시** | 왜 좋은지 이유를 대면 신뢰도 2배 | "이 이유로" "~때문에" 패턴 |
| **무료 샘플** | 무료 체험이 가장 강력한 판매 도구 | 무료 콘텐츠 → 유료 전환 구조 |
| **독자 이익** | 제품이 아닌 독자가 얻는 것에 집중 | "당신이 ~할 수 있습니다" |
| **단순성** | 쉽게 이해되지 않는 광고는 실패 | 중학생도 이해할 수 있는 언어 |

### 서비스 출력물 적용 공식
```
Hopkins식 제목 = [구체적 숫자] + [독자 이익] + [이유/근거]
예: "3분 투자로 월 300만원 더 버는 47가지 방법"
    "무릎 통증 환자 91%가 몰랐던 해결법"
```

## 2. David Ogilvy — Ogilvy on Advertising (1983) + Confessions of an Advertising Man

### 핵심 철학
> "소비자는 바보가 아니다. 그녀는 당신의 아내다."

### Ogilvy의 황금 원칙

**헤드라인의 법칙**
- 헤드라인을 읽는 사람은 본문을 읽는 사람의 **5배**
- 헤드라인에 제품 이름과 혜택을 동시에 넣어라
- 뉴스 가치가 있는 헤드라인이 가장 강력하다
- "어떻게" "왜" "새로운" "지금" "공개합니다" → 클릭률 상승 단어

**이미지/비주얼 법칙**
- 이미지는 본문보다 먼저 읽힌다
- 캡션은 본문보다 2배 더 많이 읽힌다
- 사람 얼굴이 있는 이미지 > 없는 이미지
- 제품을 사용하는 사람의 이미지 > 제품만 있는 이미지

**브랜드 이미지 법칙**
```
모든 광고는 브랜드 이미지에 기여해야 한다
단기 판매 + 장기 브랜드 = 최적 광고
```

**오길비 카피 10계명**
1. 독자에게 직접 말해라 (you/your 사용)
2. 긴 카피는 짧은 카피보다 더 팔린다 (흥미있는 내용이면)
3. 가격을 밝혀라 (숨기면 불신)
4. 보증/보장을 제공하라
5. 추천사(testimonial)를 활용하라
6. 구체적인 사실을 나열하라
7. 무료 정보를 제공하라
8. 스토리를 활용하라
9. 명령형으로 끝내라 ("지금 클릭하세요")
10. 다시 읽게 만들어라

### 서비스 출력물 적용
```
썸네일 문구 공식 (Ogilvy식):
[뉴스가치 단어] + [독자 이익] + [구체성]
"드디어 공개합니다" / "지금 밝혀진" / "이제 알 수 있는"
```

## 3. Eugene Schwartz — Breakthrough Advertising (1966)

### 핵심 철학: 욕구의 5단계 (인지 스펙트럼)

> "당신은 욕구를 만들 수 없다. 이미 존재하는 욕구를 채널링할 뿐이다."

```
1단계: 완전 무인지 (Unaware)
       → 문제 자체를 모름
       → 전략: 충격적 사실/통계로 문제 인식 유발

2단계: 문제 인지 (Problem Aware)
       → 문제는 알지만 해결책 모름
       → 전략: 공감 + "이런 방법이 있습니다"

3단계: 해결책 인지 (Solution Aware)
       → 해결책 있다는 건 알지만 어떤 상품인지 모름
       → 전략: 상품 카테고리 제시

4단계: 상품 인지 (Product Aware)
       → 우리 상품 알지만 아직 구매 안 함
       → 전략: 차별화/가격/보증 강조

5단계: 완전 인지 (Most Aware)
       → 상품 잘 알고 거의 구매할 준비
       → 전략: 가격 인하/마감/혜택 강조
```

**서비스 적용**: 풀링 콘텐츠 = 1~2단계 / 키 콘텐츠 = 3~4단계 / 랜딩페이지 = 5단계

### 매스 디자이어 공식
```
대중의 욕구 파악 → 욕구를 증폭시키는 이미지/언어 매핑 → 제품과 연결
```

## 4. John Caples — Tested Advertising Methods (1932)

### 핵심 철학: 테스트가 전부다

**역대 최고 헤드라인 사례 분석**
- "They Laughed When I Sat Down At the Piano"
  → 공감 + 변화 + 사회적 증명의 완성형
- 패턴: [조롱/의심] → [놀라운 반전] → [당신도 할 수 있다]

### Caples의 헤드라인 카테고리

| 유형 | 예시 | 효과 |
|------|------|------|
| **자기 이익** | "돈을 2배로 버는 법" | 가장 강력 |
| **뉴스** | "드디어 발견된 ~" | 호기심 유발 |
| **호기심** | "왜 어떤 사람들은 항상 ~" | 클릭 유도 |
| **빠른 쉬운 방법** | "3분 만에 ~하는 법" | 편의성 욕구 |
| **직접 명령** | "지금 바로 ~하세요" | 행동 유도 |

### 서비스 출력물 공식
```
Caples 유형별 제목 생성기:
자기이익형: "[구체적 혜택]하는 [숫자]가지 방법"
뉴스형:    "드디어 밝혀진 [주제]의 충격적 진실"  
호기심형:  "왜 [A]는 성공하고 [B]는 실패하는가"
속도형:    "[짧은 시간]에 [놀라운 결과]내는 법"
```

## 5. Joe Sugarman — Advertising Secrets of the Written Word

### 핵심 철학: 슬리퍼리 슬로프

> "카피의 유일한 목적은 다음 문장을 읽게 만드는 것이다."

**슬리퍼리 슬로프 구조**
```
헤드라인 → 첫 문장 → 두 번째 문장 → ... → CTA
각 단계는 다음 단계를 읽게 만들 뿐 → 구매는 자연스러운 귀결
```

**심리 트리거 23개 (핵심 10개)**

| 트리거 | 설명 | 적용 |
|--------|------|------|
| 일관성 | 한번 "예"라고 하면 계속 예 | "이미 해보셨죠?" 패턴 |
| 권위 | 전문가/데이터 인용 | "76% 달성률" 데이터 |
| 정당화 | 구매 이유를 제공 | "이래서 지금 사야 합니다" |
| 탐욕 | 더 많이 얻고 싶은 욕구 | "지금 사면 추가로 ~까지" |
| 희소성 | 한정 수량/기간 | "3일만 할인" |
| 단순성 | 쉽다는 느낌 | "단 3단계" |
| 인간관계 | 친밀감 구축 | "저도 한때는~" |
| 증거 | 수치/사례/연구 | 성공 사례 구체화 |
| 보증 | 리스크 제거 | "만족 못하면 전액 환불" |
| 연결 | 구매자와의 공감대 | "당신처럼 나도~" |

## 6. Gary Halbert — The Boron Letters

### 핵심 철학: 굶주린 군중 찾기

> "최고의 마케터가 되는 것보다 굶주린 군중을 찾는 게 낫다."

**굶주린 군중 (Starving Crowd) 법칙**
```
성공 방정식:
올바른 시장 + 평범한 마케팅 > 평범한 시장 + 최고의 마케팅
```

**A-pile vs B-pile 이론**
- 사람들이 메일(콘텐츠)을 두 그룹으로 나눔
- A-pile: 꼭 열어보는 것 (개인적, 손편지 스타일)
- B-pile: 나중에 볼 것 (광고 같은 것) → 대부분 버려짐
- 전략: 항상 A-pile처럼 보이게 만들어라

**서비스 적용**
```
제목/썸네일이 A-pile인가 B-pile인가?
A-pile 특성: 개인적, 대화체, 나와 관련된 느낌
B-pile 특성: 광고 느낌, 일반적, 거리감
```

## 7. Victor Schwab — How to Write a Good Advertisement

### AIDA 진화 공식

```
원래 AIDA: Attention → Interest → Desire → Action
Schwab 확장: 
Attention → Interest → Conviction → Desire → Action → Satisfaction
```

**헤드라인 100 리스트 (핵심 패턴)**

| 패턴 | 예시 |
|------|------|
| How to | "어떻게 ~하는가" |
| The Secret of | "~의 비밀" |
| Here is | "드디어 나온 ~" |
| Warning | "주의: ~하지 마세요" |
| Which of these | "이 중 어떤 것이 당신의 ~인가" |
| These mistakes | "이 실수들이 당신을 망친다" |

## 통합 카피라이팅 공식 (서비스 출력물용)

### 제목 생성 마스터 프레임워크

```
레벨 1 (기본): [주제]하는 [숫자]가지 방법
레벨 2 (Hopkins): [숫자] + [구체적 결과] + [기간]
레벨 3 (Caples): [공감 상황] + [놀라운 반전] + [독자 혜택]
레벨 4 (Schwartz): [고객 인지 단계에 맞는] + [욕구 언어]
레벨 5 (Ogilvy): [뉴스성] + [독자 이익] + [구체성] + [브랜드]
```

### 대본/원고 구조 (Sugarman 슬리퍼리 슬로프 적용)

```
1. 후크 (첫 3초): 충격적 사실 or 강한 공감
2. 문제 증폭: 이 문제가 얼마나 심각한지 (Schwartz 인지 2단계)
3. 해결책 제시: 내가 발견한 방법 (Schwartz 3단계)
4. 증거: 데이터/사례/전문가 (Hopkins 구체성)
5. 혜택 나열: 독자가 얻는 것 (Ogilvy 독자 이익)
6. 반론 처리: 의심하는 사람을 위한 답변 (Sugarman 신뢰)
7. CTA: 명확한 행동 지시 (Caples 명령형)
```

### 썸네일 문구 생성 원칙 (통합)

```
Hopkins: 숫자 + 구체성 → "3일 만에" "47%"
Ogilvy: 뉴스가치 → "드디어" "공개" "밝혀진"
Caples: 반전 공감 → "비웃었지만" "몰랐던" "후회한"  
Halbert: A-pile 느낌 → 대화체, 개인적
Schwab: 경고/호기심 → "주의" "절대" "이것만"
```
