# 프로젝트 작업 가이드 - 범용 (Common Guide)

> **목적**: 모든 코드 프로젝트에서 재사용 가능한 범용 원칙 및 규칙
> **버전**: v2.0 (웹 개발용 재구축)
> **최종 수정**: 2026-03-19
> **기반**: Streamlit 버전 PROJECT_GUIDE_COMMON.md에서 웹 개발에 맞게 업데이트

---

## 1. 진행 절차 (필수 6단계)

모든 작업은 다음 6단계를 거쳐야 함:

### 1단계: 작업 계획 설명
- 무엇을 할 것인지 구체적으로 설명
- 왜 이 작업이 필요한지 이유 명시
- 예상 결과와 성공 기준 제시

### 2단계: 현황 분석 및 확인
- 현재 상태 정확히 파악
- 관련 파일들 읽기 및 분석
- 문제점이나 개선점 명확히 식별

### 3단계: 해결 방안 제시
- 구체적인 해결 방법 설명
- 대안이 있다면 장단점 비교
- **사용자 승인 요청**: "이 방법으로 진행해도 될까요?"

### 4단계: 단계별 구현
- 각 단계마다 무엇을 하는지 설명
- 파일 수정 전후 명확히 구분
- 중요한 변경사항은 미리 설명

### 5단계: 테스트 및 검증
- 작업 완료 후 반드시 테스트 실행
- 결과 확인 및 성공 여부 판단
- 문제 발생시 원인 분석 및 재시도

### 6단계: 결과 보고
- Before/After 비교
- 달성한 성과
- 다음 작업에 대한 제안

---

## 2. 금지사항

- 설명 없이 갑작스럽게 파일 수정
- 사용자 확인 없이 여러 단계를 한번에 진행
- 테스트 없이 작업 완료 선언
- 결과에 대한 충분한 설명 없이 마무리
- `.env`, API 키, 개인정보 파일 커밋

---

## 3. 파일 네이밍 규칙

### Next.js 컨벤션
- 페이지: `app/[route]/page.tsx`
- API: `app/api/[route]/route.ts`
- 컴포넌트: `PascalCase.tsx` (예: `MemberList.tsx`)
- 유틸리티/라이브러리: `kebab-case.ts` (예: `hard-filter.ts`)
- 타입 정의: `kebab-case.ts` (예: `member.ts`)

### Development_History 파일
- 문서: `숫자-순번_[내용].md` (예: `01-01_테이블_생성.md`)
- SQL: `숫자-순번_[내용].sql`
- 결과물: `숫자-순번_Results/` 폴더

**핵심 원칙: 파일명만 봐도 이 파일이 뭐하는 건지 바로 알 수 있어야 한다!**

## 4. Git/GitHub 관리 규칙

### 커밋 메시지 규칙

**형식**: `[카테고리] 간결하고 명확한 설명`

```
Add: 새 기능/파일 추가
Update: 기존 기능 수정
Fix: 버그 수정
Docs: 문서 작성/수정
Style: UI/스타일 변경
Refactor: 리팩토링
Clean: 폴더/코드 정리
```

### 커밋 필수 시점
1. 각 Step 완료 시
2. 중요 기능 구현 완료 시
3. DB 스키마 변경 시
4. 중요 마일스톤 완료 시

### 주의사항
- 커밋 후 반드시 GitHub 푸시
- `.env.local`, API 키는 `.gitignore`에 포함
- 의미 있는 단위로 커밋

## 5. Development_History 관리

### 번호 부여 체계
```
Development_History/
└── 01_주제명/
    ├── 01-01_세부작업1.md
    ├── 01-01_Results/
    ├── 01-02_세부작업2.md
    └── 01-02_Results/
```
- 큰 주제: `01_`, `02_`, `03_` ... (연속)
- 세부 작업: `01-01`, `01-02`, `01-03` ... (각 주제 내에서 연속)

### 작업 완료 시 필수
- [ ] Development_History에 문서(md) 저장
- [ ] 결과가 있으면 Results 폴더에 저장
- [ ] Git 커밋 + GitHub 푸시

### 관리 규칙
- 작업 파일은 해당 번호 폴더에 직접 생성
- 빈 폴더 금지 (파일 없으면 삭제 또는 .gitkeep)
- 큰 작업 완료 시 SUMMARY.md 작성 **필수**

### 필수 파일 규칙 (각 주제 폴더마다)

**1. README.md (필수)**
- 각 주제 폴더(`00_`, `01_`, `02_` ...)에 반드시 `README.md` 포함
- 해당 폴더의 **모든 파일 목록** + **관련 코드 파일** + **최종 수정일** 기록
- 파일이 추가/수정될 때마다 README.md도 같이 업데이트

```markdown
# 01_주제명 — 파일 목록
> **최종 수정**: 2026-03-21

| 파일 | 내용 | 최종 수정 |
|------|------|----------|
| 01-01_세부작업.md | 설명 | 2026-03-21 |
| 01_SUMMARY.md | 총정리 | 2026-03-21 |

**관련 코드 파일**: `lib/example.ts`, `app/api/example/route.ts`
```

**2. SUMMARY.md (Step 완료 시 필수)**
- 해당 Step 전체의 총정리
- 구현된 코드, 검증 결과, 버그 수정, 알려진 한계 등 핵심 포함
- 너무 요약하지 말 것 — 외부인이 읽고 이해 가능해야 함

**3. 코드 파일은 번호 붙이지 않음**
- Development_History 문서: `숫자-순번_[내용].md` (번호 있음)
- 실제 코드 파일: `kebab-case.ts` (번호 없음, Next.js 컨벤션)
- 코드 파일은 README.md의 "관련 코드 파일" 테이블에서 추적

**4. 최종 수정일 기록**
- 모든 문서와 README에 `최종 수정: YYYY-MM-DD` 표시
- 파일 수정 시 날짜도 같이 업데이트

**5. 테스트 파일 관리 (필수)**
- 테스트/검증용 임시 파일 (`test-*.js`, `scripts/` 등)은 **검증 완료 후 즉시 삭제**
- 검증 결과는 md 문서에 기록하되, 테스트 코드 자체는 로컬+git 모두에서 제거
- `.gitignore`에 `test-*.js`, `scripts/` 포함되어 있어 실수로 커밋 방지
- 이유: 임시 파일이 쌓이면 프로덕션 코드와 혼동, 프로젝트가 정리 안 된 인상

## 6. 새 터미널 세션 시 지시사항 템플릿

```
프로젝트 계속 작업해줘.

=== 현재 프로젝트 정보 ===
프로젝트명: [프로젝트명]
작업 디렉토리: [경로]

=== 폴더 구조 ===
Development_History/
├── 00_[주제]/ (완료)
├── 01_[주제]/ (현재 작업중)
└── ...

=== 완료한 작업 ===
1. [작업 내용]

=== 다음 작업 ===
1. [다음 작업]

이 구조로 계속 진행해줘.
```

## 7. 추가 규칙 (진행하면서 업데이트)

> 프로젝트를 진행하면서 발견되는 새로운 규칙이나 패턴은 여기에 추가합니다.

### 7-1. 문서 관리 규칙 (설계 변경 시)

브레인스토밍 → 구현 → 리뷰 → 리팩토링을 거치면 구버전 문서가 쌓인다. 아래 규칙으로 관리:

**1. 설계서.md = 현재 기준 (Single Source of Truth)**
- 각 주제 폴더에 `XX_[주제]_설계서.md` 형태로 생성
- 설계 변경 시 **이 문서를 먼저 업데이트**하고, 코드를 맞춤
- 팀원이 "지금 어떻게 돌아가지?"를 알려면 이 파일만 보면 됨

**2. 번호 파일 (XX-01, XX-02...) = 히스토리 (과정 기록)**
- 작업 당시의 상태를 보존. **수정하지 않음.**
- 나중에 "왜 이렇게 결정했지?" 추적할 때 사용
- 설계 변경으로 구버전이 된 경우 **상단에 DEPRECATED 표시 + 최신 문서 링크**:
```
> ⚠️ DEPRECATED: 이 문서는 구버전입니다.
> 최신 설계는 `XX_[주제]_설계서.md`를 참조하세요.
```

**3. SUMMARY.md = 폴더 전체 요약 (항상 최신)**
- 설계서 변경 시 SUMMARY도 같이 업데이트
- 폴더 내 모든 파일의 역할과 현재 상태를 한눈에 보여줌

**정리하면:**
```
XX_[주제]/
  ├── XX_[주제]_설계서.md     ★ 현재 기준 (항상 최신)
  ├── XX_SUMMARY.md           ★ 폴더 요약 (항상 최신)
  ├── XX-01_[작업].md          히스토리 (수정 안 함)
  ├── XX-02_[작업].md          히스토리 (구버전이면 DEPRECATED)
  └── XX-0N_MAIN_Results/      결과물/백업
```
